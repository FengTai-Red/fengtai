搭建JAVA环境

首先先去装一个Java Extension Pack扩展，这个里面包含写JAVA需要用到的许多其他扩展。

然后再装一个Eclipse New Java Project，然后按下面的步骤操作

1.选择一个工作区,就是项目要放的文件夹

2.按下 Shift+Ctrl+P进入命令面板，输入New Java Project 创建项目

3.输入项目名

4.选择JDK版本

上面创建完项目后，我们看到在项目文件夹中有两个子文件夹，一个用来存源码即src，一个用来存class文件即bin，另外还有两个文件一个是.classpath文件，一个是.project文件

代码文件的话放在src文件夹下，里面可以建文件夹，相当于包。代码写完后按F5就可以运行了

导入第三方包

再学习过程中又遇到了阻碍，看的教程是使用Eclipse，但我使用VSVode的，于是又摸索了一番，以下为步骤：

1.在项目下再建立一个文件夹，和src、bin等同级，命名为lib。同时在lib下再建立一个名为sources的文件夹(不要源码也行)

2.将要引入的jar包复制到lib目录下，源码文件包复制到sources文件夹下(不要源码也行)

3.修改.classpath文件，增加一行classpathentry，并填入正确的路径，可以填相对路径，也可以填绝对路径

<classpathentry kind="lib" path="lib/commons-io-2.6.jar" sourcepath="lib/sources/commons-io-2.6-sources.jar"/>

上面代码的sourcepath属性为源码，若不导入源码只需将sourcepath属性删除即可

The local variable * may not have been initialized // 局部变量*可能未初始化(1.局部变量 2.没有赋值(建议赋值null))

CompileError

* cannot be resolved or is not a field // *无法解析或不是字段

This method must return a result of type * // 这个方法必须返回一个**类型

Collection is a raw type. References to generic type Collection should be parameterized // 容器指定类型

The left-hand side of an assignment must be a variable // 赋值左侧必须为变量

JAVA打JAR包

用的是VSCode，不能像Eclipse那样直接打包，只能通过CMD来打包了。

VSCode的终端实际上是个引用了系统的CMD,只不过它自动帮我们找到了当前的文件夹。所以直接在VSCode中操作就行了。

首先在项目目录(和src同以目录)下新建一个清单文件：manifest.txt文件。里面需要手动输入一些信息。

Manifest-Version: 1.0

Class-Path: lib/jarA lib/jarB

Main-Class: Demo.Test

 

上面一共有四行，作用分别如下：

1.Manifest-Version: //指定Manifest的版本，这里不用改

2.Class-Path: //导入程序引用的外部包的路径，多个使用空格隔开，没有可以不用写。

3.Main-Class: // 告诉jar包中要运行的类，这里是运行src文件夹下的Demo文件夹下的Test类

4.//空行。最后一定要有这个空行，否则创建好的JAR不会把主类写进清单文件里

写好manifest文件后，就在VSCode的终端输入以下命令：

jar -cvfm name.jar manifest.txt -C bin/ .

其中name就是打包后JAR包的名字，可以自行修改

随后就可以在项目的目录下看到生成的JAR包了。
